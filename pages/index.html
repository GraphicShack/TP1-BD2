Présentation du projet de session — SMI1002 (Hiver 2026)

Dans le cadre de ce projet de session, on devait concevoir et implémenter une base de données relationnelle avec Oracle, basée sur un problème concret choisi. L’idée n’était pas seulement de créer des tables, mais vraiment de travailler dans un contexte plus réaliste, où la base de données est hébergée sur un serveur et accessible par plusieurs utilisateurs en même temps. Ça nous a permis d’aborder des notions importantes comme la concurrence, la gestion des transactions et la cohérence des données.

Le projet est divisé en deux grandes parties. D’abord, on a travaillé sur la conception de la base de données : définir le problème, analyser les besoins et créer un modèle complet avec les entités, les relations, les clés primaires et étrangères, ainsi que les règles de gestion. Ensuite, on a implémenté cette structure dans Oracle.

Dans la deuxième partie, on s’est concentrés sur les aspects plus techniques, comme l’optimisation des requêtes, la gestion des accès simultanés, les transactions, l’indexation et la récupération des données en cas de panne. On a aussi intégré des interfaces simples pour permettre d’interagir avec la base de manière claire et logique.

Au niveau des outils, on utilise principalement Oracle et SQL Developer, et pour l’interface, on peut choisir le langage qu’on veut, tant que ça fonctionne sur les machines de l’université.

Finalement, ce projet demande beaucoup d’autonomie et de rigueur, parce qu’on doit être capables d’intégrer plusieurs concepts importants vus dans le cours, comme :

  • la modélisation des données
  • l’intégrité référentielle
  • les transactions (ACID)
  • la gestion de la concurrence
  • l’optimisation et les index
  • les objets Oracle (procédures, vues, séquences)
  • la journalisation des opérations
  • la gestion des pannes
